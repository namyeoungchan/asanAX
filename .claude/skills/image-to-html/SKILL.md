# image-to-html — 이미지·PDF 픽셀 분석 → HTML 변환

이미지를 작은 격자로 다운샘플링하면 각 칸의 '평균 색'이 나옵니다.
그 색들을 CSS 그리드 블록으로 깔면, 원본을 닮은 색상 모자이크 HTML이 됩니다.
여기에 가장 자주 나오는 대표 색 팔레트와 원본 크기 정보를 함께 담아
의존성 없는(self-contained) 단일 `.html` 파일로 출력합니다.

## 필요 패키지
- **Pillow** (필수): `pip install pillow`
- **PyMuPDF** (PDF 입력 시): `pip install pymupdf`  ← 없으면 이미지 입력만 처리

## 사용법
```bash
python convert.py <입력파일> [-o 출력.html] [--cols 96] [--palette 6]
```
- `<입력파일>` : `.jpg .jpeg .png .webp .bmp .gif` 또는 `.pdf`
- `-o`        : 출력 HTML 경로 (기본: 입력파일명 + `.html`)
- `--cols`    : 가로 격자 칸 수 = 해상도(클수록 정밀, 무거움). 기본 96
- `--palette` : 대표 색 개수. 기본 6

## 동작 방식 (픽셀 단위)
1. 이미지를 RGB로 열고(EXIF 회전 보정), `--cols` 너비로 비율 유지 축소
   → 축소가 곧 '구역 평균색' 계산이 됩니다.
2. 각 픽셀의 RGB를 읽어 한 칸(`<i>`)의 배경색으로 깔아 모자이크 생성.
3. 전체 픽셀을 양자화해 빈도순 대표 색 N개(팔레트)를 추출.
4. 원본 크기·평균색·팔레트·모자이크를 담은 단일 HTML로 저장.

## 한계
- 글자 인식(OCR)이나 레이아웃 복원은 하지 않습니다 — '색/구조의 시각화'예요.
- 칸 수(`--cols`)를 키울수록 정밀하지만 파일이 커집니다(권장 64~160).
- 텍스트 문서 구조 분석은 `doc-analyzer` 스킬을 쓰세요.
