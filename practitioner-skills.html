THE BIG PICTURE · 현업의 일하는 법

주방장은
혼자 다 하지 않아요

바쁜 식당의 주방장은 모든 요리를
맨손으로 처음부터 하지 않습니다.

· 레시피북(스킬)을 펼쳐 정해진 절차대로,
· 큰 주문은 전담 요리사(에이전트)에게 통째로 맡기고,
· 배달·결제·재고(MCP)는 외부 업체와 연결해 둡니다.

Claude도 똑같아요. 이 세 가지를 알면
‘대충 시키는’ 수준에서 ‘현업처럼 부리는’ 수준으로 올라갑니다.

KEY IDEA · 스킬

스킬은 ‘분야별 전문 레시피북’이에요

그냥 “이거 해줘”라고 하면 Claude는 매번 즉흥적으로 만듭니다.
스킬은 “이 작업은 이런 순서·기준으로 한다”를 적어둔 전문 매뉴얼이에요.
해당 작업을 만나면 Claude가 그 매뉴얼을 펼쳐 똑같이 높은 품질로 처리합니다.

현업에선 ‘회사의 일하는 방식’을 스킬로 만들어요

코드 리뷰 기준, 디자인 톤, 배포 절차처럼 ‘우리 팀은 이렇게 한다’를 스킬로 적어두면, 누가 시키든 같은 기준으로 결과가 나옵니다.

# SKILL.md — 스킬의 ‘표지’ # 이 부분을 보고 언제 쓸지 판단해요 --- name: jsp-to-react description: JSP를 리액트로 전환할 때 사용. 'JSP 전환', '리액트로' 라고 하면 작동 --- # ↓ 본문엔 실제 절차가 단계별로 # 1) 컴포넌트 추출 2) API 매핑 …

description이 핵심 — 여기 적힌 상황을 만나면 스킬이 자동으로 불려와요

HOW · 스킬 정보를 활용하는 법

스킬의 ‘설명’을
읽는 법부터

모든 스킬엔 SKILL.md 라는 안내서가 있어요.
맨 위 description이 “언제 이 스킬을 쓰는지”를 정해줍니다.

  1. 찾기 — “이런 거 하는 스킬 있어?”라고 물으면 find-skills가 맞는 걸 찾아줘요
  2. 읽기 — 후보 스킬의 description으로 용도를 확인
  3. 부르기 — 그 상황에 맞는 말("리액트로 전환해줘")만 하면 자동 적용
REAL SKILLS · 이 강의 폴더에 진짜로 들어있어요

말로 부르면 파이썬이 도는, 실전 스킬 두 개

개념만 말고 진짜를 넣어놨어요.
이 강의 저장소의 .claude/skills/ 안에 동작하는 스킬 두 개가 있습니다.
코드를 몰라도 됩니다 — 상황에 맞는 말만 하면 Claude가 알아서 실행해요.

AGENT · 에이전트 활용법

큰 일은 ‘전담 직원’에게 통째로 맡겨요

스킬이 ‘매뉴얼’이라면, 에이전트는 그 매뉴얼을 들고 독립적으로 일하는 직원이에요.
큰 작업을 떼어 맡기면, 알아서 여러 단계를 처리하고 결과만 보고합니다.
그동안 나(메인)는 다른 일을 할 수 있어 ‘분업’이 됩니다.

스킬·에이전트·MCP의 차이

스킬은 ‘일하는 방법(매뉴얼)’, 에이전트는 ‘일을 맡는 사람’, MCP는 ‘바깥 세상과 이어주는 통로’예요. 셋을 합치면 “조사하고 → 만들고 → 노션에 정리”까지 한 흐름으로 됩니다.

현업자가 꼭 챙기는 세 가지

도구가 강력할수록, ‘언제 무엇을 쓸지’와 ‘안전’을 같이 챙겨야 해요.

①
큰 일만 에이전트에게
짧은 일까지 맡기면 오히려 느려요. ‘넓게 뒤지거나 독립적인 일’만 분업.
②
MCP 연결엔 권한이 따라와요
노션·메일에 접근하는 권한이라, 회사 계정·민감 데이터는 신중히. 필요한 것만 연결.
③
정리는 그때그때
작업 끝날 때 바로 노션에 남기면, 나중에 회고·인수인계가 공짜로 끝납니다.
정리 끝, 직접 해보기

이제 ‘시키는’ 게 아니라 ‘부려’ 봅시다

스킬로 일관되게, 에이전트로 분업하고,
MCP로 외부 서비스까지 — 한 흐름으로 묶는 게 현업의 일하는 법이에요.
